# Clean Branches - マージ済みローカルブランチ削除

## 概要

developブランチにマージ済みのローカルブランチを安全に削除する。
以下のブランチは削除対象から除外する：

- `main`
- `develop`
- `release/*` ブランチ
- 現在チェックアウト中のブランチ

## 実行手順

### 1. developブランチを最新化

```bash
git fetch origin
git checkout develop
git pull origin develop
```

### 2. マージ済みブランチの一覧取得

```bash
git branch --merged develop
```

### 3. 削除対象の特定

上記の結果から以下を除外：
- `main`
- `develop`
- `release/*` パターンに一致するブランチ
- `*` マークがついている現在のブランチ

### 4. ユーザーに確認

削除対象のブランチ一覧をユーザーに提示し、確認を求める。
対象がない場合は「削除対象のブランチはありません」と伝えて終了。

### 5. 削除実行

確認後、対象ブランチを `git branch -d` で削除する。

```bash
git branch -d <branch-name>
```

### 6. 結果報告

削除したブランチと残っているブランチの一覧を報告する。

## 注意事項

- 必ず `git branch -d`（小文字のd）を使用し、マージ確認付きで削除する
- `git branch -D`（大文字のD）は使用しない（未マージブランチの強制削除を防ぐ）
- 削除前に必ずユーザーに確認を取る
- リモートブランチは削除しない（ローカルのみ）
