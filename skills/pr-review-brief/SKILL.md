# Human PR Review Brief 生成
PR を作る前に、人間レビュアーが最小限の確認で判断できる要約を生成する。

## 使い方
1. ブランチの差分（`git diff --name-only main...HEAD` 等）と、実行済みチェック・レビュー結果を集める。
2. 下記テンプレを埋める。埋められない欄は「該当なし/未実施」と明示する。
3. 出力は **PR 本文**に置くのが既定（証跡が人間のレビュー場所に残り、PR クローズで自動アーカイブ）。
   プロジェクトの慣習があればそれに従う。

<template>

  # 概要
  このPRで実現したこと。

  ## 変更範囲
  対象モジュール/ファイル群。

  ## 受け入れ条件との対応
  | 受け入れ条件 | 実装 | テスト |
  |---|---|---|

  ## 重要ファイル
  - ...

  ## リスク領域
  - 認証: あり / なし
  - 認可: あり / なし
  - データベース: あり / なし
  - マイグレーション: あり / なし
  - 外部API: あり / なし
  - シークレット: あり / なし
  - UI / パフォーマンス: あり / なし

  ## 実行したチェック
  - <プロジェクトのチェック名>: 成功 / 失敗 / 未実行

  ## レビュー結果
  - code-reviewer: 合格 / 注意付き合格 / 要修正 / 不要
  - security-reviewer: ...
  - db-reviewer: ...

  ## 人間が確認すべきこと
  - ...

  ## 既知のリスク / 受容したリスク
  - ...

  ## 未対応
  - ...

</template>
