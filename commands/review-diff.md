現在の作業差分を汎用的にレビューしてください。

- `@code-reviewer` を起動し、`git diff` / `git diff --cached` を対象に精査させる。
- 重大な不具合・要件/受け入れ条件との不一致・既存挙動の破壊に集中する。
- 差分が認証/DB/パフォーマンス等に触れる場合は、該当する専門レビュアー（`@security-reviewer` / `@db-reviewer` 等）の追加起動を提案する。
- 出力は統一フォーマット（判定 / ブロッキング issue / 指摘 / 再レビュー要否）。修正はメインスレッドで行う。
